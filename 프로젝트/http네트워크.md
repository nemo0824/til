캐시란? 

캐시는 데이터나 리소스를 빠르게 재사용하기 위해 임시로 저장해두는 메모리 공간 또는 저장소입니다

캐시의 주요 목적은 더 빠른 응답 시간과 효율적인 자원사용을 제공하는데 있습니다 

캐시의 기본 개념

1. 임시저장
    1. 자주 사용되거나 재사용 가능성이  높은 데이터를 캐시에 저장해 필요할 때 더 빠르게 가져올수있도록함
2. 속도향상
    1. 캐시는 cpu, RAM , 디스크 등 보다 빠른 메모리 계층에 데이터를 저장하므로 성능을 크게 향상시킨다 
    2. 예) 브라우저는 자주 방문하는 웹페이지를 로컬에 저장해 빠르게 로드합니다
3. 자원절약
같은 데이터를 반복적으로 계싼하거나 다운로드하는 비용을 줄입니다

캐시가 사용되는 주요영역

1. 웹 브라우저와 웹 서버 
2. 운영체제
3. 데이터베이스
4. api 및 네트워크 

캐시의 기본 동작 방식

1. 요청
    
    데이터를 요청 할때 먼저 캐시를 확인
    
    캐시에 데이터가 있으면 (cache hit) 데이터를 바로 반환
    
    없으면 (cache miss) 원본 데이터 소스에서 가져와 캐시에 저장 후 반환
    
2. 유효기간
    
    캐시는 유효기간이 (time-to-live TTL)이 설정되어있어서 일정시간이 지나면 데이터가 만료
    
    유효기간이 지나면 새 데이터를 가져와 캐시를 갱신
    

캐시동작

1. 사용자가 웹 사이트에 방문 브라우저는 =⇒ html, css, js 등을 캐시에 저장
2. 동일한 웹사이트 방문시 브라우저는 서버에 요청하지않고 캐시에서 리소스 로드
3. 서버는 캐시가 최신인지 아닌지 확인을 위해 ETag LastModified 헤더로 데이터 변경 여부 체크

캐시 관련된 HTTP 헤더

1.Cache-Control

캐시 동작을 제어하는 헤더 

Cache-Control : max -age=3600

2.ETag

서바가 리소스의 고유 식별자를 제공해 변경 여부를 확인

3.Expires

캐시 유효기간을 명시적으로 설정

4.Last-Modified

리소스의 최종 수정 시간을 기반으로 캐시 유효성 확인

캐싱에 대한 질문?

=⇒ 캐싱이란 웹 관점에서 봤을떄 자원소모와 빠른접근을 가능하게해준다 

하지만 캐싱을 했을때 동적인화면 즉 정보를 계속 변경해야하는 화면에서는 캐싱이될까? 라는생각을했을때
”hello”란걸 검색하는 화면이면 캐싱할 필요는없다 
하지만 검색을 반복해서 할 경우 캐싱은 사용하기도한다

캐싱에 적합한 화면은 변화가 적고 반복적으로 사용되는 리소스들이다 

1.정적콘텐츠 (이미지, 폰트 , 로로고, css스타일)

2.블로그나 뉴스기사 =⇒ 기사가 작성된이후에 내용이 거의 바뀌지않기때문에 

3.제품목록 페이지 =⇒ 제품목록은 번번히 변경이되지않음

4.회사소개홈페이지

실전활용에있어서 사용할수있는방법

1.혼합 

캐싱가능한요소(html, css)를 로드하고, 실시간 데이터 (주식, 코인)를 비동기적으로 업데이트 

2.캐싱 정책 설정(http 헤더)

Cache-Control max-3600 

3.캐시 무효화

데이터가 변경될때 캐시를 무효화해 최신 데이터를 제공
